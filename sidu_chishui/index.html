现场讲解顺序

  1. 先看地形：川南、黔北、滇东交界，山地河谷把大兵团协同变成难题。
  2. 再看渡口：太平渡、二郎滩、茅台、乌江与金沙江渡口决定行动窗口。
  3. 最后看决策：每一次渡河都是对敌情、地形、后勤和政治协同的重新排序。

三张附件地图如何组成“内置沙盘地图”

本版把三张附件地图与原创示意图放进同一个离线地图工作台，不做外链、不依赖 API、不缓存第三方地图瓦片。实际使用时按“广域—核心—战役箭线—原创推演”的顺序对读。

1. 地形广域图

用于看川南、黔北、滇东、重庆方向的大空间关系：为什么红军的南北东西转换会迫使国民党军多路调兵，为什么大范围地形起伏会放大时间差。

2. 地形核心图

用于看赤水、习水、仁怀、遵义之间的山地道路、河谷、县城与关隘，帮助解释土城、茅台、桐梓、娄山关、遵义等节点为什么成为行动枢纽。

3. 战役示意图

用于核对一渡、二渡、三渡、四渡、南渡乌江、入滇、渡金沙江的整体路线，以及红军与国民党军行动方向的对照。

4. 原创动态沙盘

用于培训推演和汇报演示：它不追求测绘精度，而把河流、渡口、决策、敌我态势和行动后果压缩到一张可讲、可点、可复盘的管理地图中。

使用边界：前两张地形截图和第三张战役示意图均为本会话用户提供附件。本成果包按内部学习研究、课堂沙盘与组织培训用途内置。第三张图面可见“中国地图出版集团制”和审图号，公开出版、商用展陈、二次分发或外部宣传前，应另行核验授权和引用边界。

GitHub Pages / 手机 PWA 部署说明

仓库根目录

把 ZIP 解压后的全部文件放在仓库根目录，保留 index.html、manifest.webmanifest、service-worker.js、assets/ 和 .nojekyll。

PWA安装

通过 HTTPS 的 GitHub Pages 打开后，Chrome / Edge / Android 浏览器可安装；iPhone Safari 可“添加到主屏幕”。本地测试建议用 python -m http.server 8000。

缓存刷新

如果旧版本仍显示百度/Google按钮，请在开发者工具 Application → Service Workers 中注销旧缓存，或修改文件后等待浏览器刷新 Service Worker。
