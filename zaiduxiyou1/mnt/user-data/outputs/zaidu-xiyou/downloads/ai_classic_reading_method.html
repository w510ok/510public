AI 辅助名著深读六步法

一套用国内大模型深读古典名著的可复用方法。换一本书、换一个底本即可照做。

标准：专业、务实、可考据、可落地。

一

建底本

确定一个可检索底本，记录版本、整理者、卷/回/篇目结构与引用边界。正式发表前保留纸质校注本复核机制。

二

建表

至少建立六张表：

  1. 主题表（章节范围 / 核心事件 / 主题 / 一句话观点）
  2. 人物关系表（人物 / 身份 / 所属阵营 / 关系对象 / 关系性质）
  3. 关键情节表（事件 / 出处回目 / 性质 / 冲突或风险类型）
  4. 对象分类表（人物、势力、物件，按来源或类型归类）
  5. 原文摘录表（回目 / 短摘≤25字 / 定位 / 主题 / 解释 / 用途 / 来源）
  6. 专题观点表（专题 / 核心论点 / 支撑情节 / 可延伸方向）
三

做图谱

把数据表转为三类图：人物关系图、风险/后台矩阵、主题热力图。图谱服务理解，不为炫技。建议先产出“可直接画图的结构化数据”（节点/边列表、矩阵表、分级评分），再交给图表工具呈现。

四

摘原文

每条摘录包含：回目、短摘、定位、主题、解释、用途、来源。以短摘为主，避免大段复制原文。凡记不准的原文一律标注“原文待核”，不得凭印象编写。

五

写专题

从数据表和摘录表中抽取观点，先写小专题，再汇成研究报告。单篇结构建议：一句话核心论点 → 从原文情节切入（标注回目）→ 2–3 个分论点（每个配 1 条短摘佐证）→ 一个现实可迁移的洞见。

六

做发布包

形成导览网页、数据底座、研究报告、公众号连载稿、讲稿与方法模板。发布前进行四类校核：

  • 口径：表述是否前后一致；
  • 引用：短摘是否标注出处、是否超出短摘边界；
  • 版权：是否含不宜公开内容、是否仅作评论/研究/索引用途；
  • 事实：情节 / 回目 / 人物是否经人工复核。

推荐迁移对象

配套提示词

《再读西游》导览页内置“国内大模型提示词工作台”，把以上六步分别封装为可直接复制、可换书的提示词，兼容 DeepSeek、Kimi、通义千问、文心一言、智谱清言、豆包等。提示词只在本地生成，模型输出仍需人工校核后方可对外发布。
