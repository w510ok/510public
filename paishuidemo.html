江南水务专业调度系统 · Demo 使用指南

本 demo 是三个核心场景的单文件原型，所有数据均为模拟，用于评审、路演与培训演练。

网格化调度管控 · Tab 1

  • 15 个网格：主城区 10 格 + 沿江 5 格。点击任一网格查看详情并可派发巡检。
  • 5 泵站 / 3 闸门 / 8 排口：一键启停机组、调整开度；沿江排口在高潮位时会闪烁提示倒灌。
  • 告警中心：规则引擎自动识别雨强 / 水位 / 潮位告警；点击条目确认。
  • 系统建议：绿色胶囊为自动生成的调度建议，点击即采纳、自动执行。
  • 响应等级：Ⅰ~Ⅳ 级，随红橙网格数自动联动。

员工岗位驾驶舱 · Tab 2

  • 3 种一线角色：网格巡检员 / 泵站班组 / 应急抢险队，顶部下拉切换。
  • 巡检员：待办任务列表、导航 / 签到 / 上报。
  • 泵站班组：我的泵站机组状态、本站指令回溯。
  • 应急抢险：队伍在岗、红色网格分布、视频接入清单。

培训演练模拟 · Tab 3

  • 3 个剧本：典型暴雨 / 暴雨+高潮位复合 / 局部强降水。
  • 沙盘时钟：1x / 2x / 4x / 8x 倍速。2 小时剧本在 4x 下约 30 秒跑完。
  • 评分规则：按时命中剧本期望动作 +10，超时 +5，剧本明确错误动作 -8，额外预置动作 +3。
  • 剧本结束自动结算，显示命中率、错误数与明细。

技术边界

  • 纯前端，单文件 HTML，无后端、无数据库、无登录。
  • 断网可用（ECharts 自动降级为内置 SVG 线图）。
  • 所有调度指令写入 STATE.commands 审计流（带 traceId，可在 console 查）。

调试入口

  • STATE — 全局状态
  • clock.start() / pause() / stop() — 时钟控制
  • startPump('P01') / stopPump('P01') — 手动调度
