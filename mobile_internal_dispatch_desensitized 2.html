1 汛情研判
先看风险，再看点位，再安排调度
当前重点风险：暴雨内涝、沿江高水位 / 潮位顶托、低洼片区积水、下穿通道与地下空间倒灌、泵站高负荷或故障、供排水设施停电、井盖冒溢与污水外溢，以及“天气 + 设备 + 用户报障”叠加形成的复合型险情。

复合灾害

不是单点事故，而是暴雨、顶托、外电波动、交通受阻和群众报障同时出现的连续场景。

全链条治理

调度对象已从单一水厂或泵站，扩展为“厂—站—网—河—用户”的统一事件处理链。

关键设施优先

关键泵站、原水取输、水厂高配电、重点排口、历史积水点、下穿通道应先纳入统一清单。

人机协同

先做“规则阈值 + 模型辅助 + 值班确认”的轻量闭环，比一开始追求复杂大系统更适合落地。

4 调度重点速览
值班手机端先看这四类

先盯天气与外河

短临强降雨、外河高水位和顶托预警是多数处置动作的起点，应与值守、封控和预置同步启动。

再盯泵站与排口

液位接近上限、备用泵不可用、通信中断、外电波动时，优先升级事件等级。

同步盯群众报障

同一网格短时间内出现多起同类报障，可视为“群众传感器”，用于校验是否形成积水或停水点。

最后盯回传复盘

到场时间、恢复时长、是否顶托、是否停电、是否舆情集中，应在事件结束后快速沉淀。

✓

预警前移

联看气象短临预报、外河水位 / 潮位、泵站液位、重点用户报障和历史积水点；满足升级条件时不等待人工补报。

✓

设施巡查

水厂、泵房、闸站、排口、井盖、下穿通道、高配电室、机房和施工点位按“问题—责任—时限—复核”闭环巡检。

✓

物资前置

移动泵车、发电机、沙袋、挡水板、警示牌、排水软管和照明设备围绕历史高风险点前置到位。

✓

联调联排

供水、排水、客服、抢修、物业、街道、防办、供电同图会商、同网格派单，避免“看得见但调不动”。

✓

安全作业

涉水涉电执行“先断电、后作业；两人同行、先封控后抢险”；地下空间和交通要道先做风险隔离。

✓

保供保排

供水侧紧盯水质、水压、原水异常与关键用户连续性；排水侧盯液位、满流率、排口回水和污水厂高负荷运行。

✓

抢修复电

关键泵站和厂站配电室要做到移动电源快速接入、远程 / 现场双启动、抢修过程留痕并同步更新用户通知。

✓

处置复盘

重大事件 48 小时内完成复盘，记录积水时长、到场时长、恢复时长、是否顶托、是否停电、是否舆情聚集。

✓

收到预警减少非必要外出

避开低洼路段、沿江低地、施工围挡周边和已发布封控提示的道路。

✓

远离下穿通道、地下车库、井盖冒溢区域

不围观、不涉水试探，不在雨水口和井盖附近停留。

✓

车辆不要强行涉水

涉水熄火后不要二次启动；电动车和充电设备远离积水区域。

✓

发现险情及时拍照定位上报

积水、污水外溢、井盖缺失、供水异常、路面塌陷等情况，应尽快通过统一入口上报。

蓝色预警

关注局地强降雨、短时积水和个别点位液位抬升。

  • 员工：加强巡查，核实重点泵站、排口、下穿通道状态。
  • 市民：留意预警信息，减少前往低洼地段。

黄色预警

降雨量级提升，局部风险点可能进入快速积水阶段。

  • 员工：队伍前置，历史易涝点和沿江顶托敏感点值守。
  • 市民：避开地下空间和积水路段。

橙色预警

暴雨叠加外河高水位或关键设施故障，需进入强排与封控阶段。

  • 员工：启动强排、临时封控、重点设施保护和移动电源接入。
  • 市民：非必要不外出，服从交通管制和社区提醒。

红色预警

超标准降雨或复合型灾害来临，优先保障人身安全。

  • 员工：先人后物，优先保障生命线设施。
  • 市民：立即避险转移，不涉水抢行。
